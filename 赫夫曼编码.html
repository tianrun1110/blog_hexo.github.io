赫夫曼编码

发表于 2020-08-25 更新于 2020-08-27 热度： ℃ Valine：
本文字数： 5.7k 阅读时长 ≈ 5 分钟

赫夫曼编码是电讯通信中的经典应用之一。
赫夫曼编码用于数据文件压缩。其压缩率通常在20%～90%之间。
赫夫曼编码是可变字长编码（VLC）的一种。

例子：

i like like like java do you like a java
将字符的个数对应起来：d:1 y:1 u:1 j:2 v:2 o:2 l:4 k:4 e:4 i:5 a:5 (空格):9
按照上面字符出现的次数构建一颗赫夫曼树，次数为权值。
链接：构建赫夫曼树

根据赫夫曼树，给各个字符规定编码，向左的路径为0，向右的路径为1，如上图所示。
编码如下：
o:1000 u:10010 d:100110 y:100111 i:101 a:110 k:1110 e:1111 j:0000 v:0001 l:001 (空格):01
按照赫夫曼编码，i like like like java do you like a java的编码为（无损压缩）：
10101001101111011110100110111101111010011011110111101000011000011100110011110000110011110001001001001101111011100100001100001110 长度为133
不压缩长度为：359
注意：
赫夫曼树根据排序不同，也可能不太一样，这样对应的赫夫曼编码也是不完全一样的，但是wql是一样的，并且都是最小的。
